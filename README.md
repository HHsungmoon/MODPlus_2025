# MODPlus_2025

## 🧪 프로젝트명

**MODplus의 비제한적 탐색 (Unrestrictive Search) 성능 향상을 위한 멀티스레드 구현**

---

## 📄 프로젝트 요약

소프트웨어 성능 최적화에 흥미가 있어 성능 개선 프로젝트를 진행하길 결심하고 팀을 이루었다.  
프로젝트의 목적은 기존의 코드를 멀티스레드로 구현하여 **동일한 결과를 산출하면서도 수행 시간이 훨씬 단축**되도록 만드는 것이다.
해당 과제의 목적이 관심 분야와 일치해 프로젝트를 진행하게 되었으며, 앞으로 수행 시간을 **획기적으로 개선하여 대용량 데이터 분석에 적합한 구조로 리팩토링**하는 것이 목표이다.

기존 대비 **30배 이상의 실행 속도 개선**을 달성한 **최적화된 병렬 프로그램**으로,  
**오류 및 버그 없이 안정적으로 구현**하고자 한다.

## 1. 프로젝트 배경 및 목표

### ■ 프로젝트의 필요성
- 현재까지도 **2015년도 코드**를 사용 중이며, 단일 스레드 기반이라 **실행 속도가 매우 느림**
- **분석 데이터 규모는 계속 증가**하고 있으나, 기존 구조로는 처리에 한계 존재
- 멀티스레드 리팩토링을 통해 **본래 기능과 탐색 정확도를 유지**하면서도 수행 시간을 **대폭 단축** 가능
- 연구실 내 다양한 과제에서의 **실제 활용도 향상**이 기대됨
- github : https://github.com/HanyangBISLab/MODplus
---

### ■ 프로젝트의 정의
본 프로젝트는 단백질의 **Post-Translational Modification (PTM)** 분석을 효과적으로 수행할 수 있는 도구인 **MODplus**의 성능을 개선하기 위한 최적화 프로젝트입니다.  
Java로 구현된 기존 단일 스레드 기반 코드를 **멀티스레딩** 구조로 재구현하여 성능을 최대한 향상시키는 방식으로 리팩토링을 진행합니다.

특히 **Unrestrictive Search** 방식에서의 성능 병목 현상을 해결하기 위해 알고리즘 최적화,다중 스레드 기반 병렬화를 적용합니다.

또한 MODplus 실행 결과의 **정확도 일관성**과 **스레드 간 동시성**을 보장하며, 최신 JDK 및 라이브러리를 활용한 **모듈화**, `Dynamic Programming` 기반 알고리즘과 자료구조의 최적화도 함께 수행합니다.

---

### ■ 프로젝트 최종목표
- **Unrestrictive Search 수행 시간**을 획기적으로 개선
- **대용량 데이터 분석에 최적화된 구조**로 리팩토링
- **탐색 정확도 및 결과 정합성 유지**
- **CPU 개수에 비례한 성능 확장성 확보**

---

### ■ 세부목표
1. 기존 MODplus 코드 분석 및 멀티스레드 적용 가능한 부분 식별  
2. Java의 `Thread`, `ExecutorService`, `ForkJoinPool` 등을 활용한 병렬 처리 구조 설계  
3. 기존 클래스 및 함수 최적화, 적절한 자료구조 적용  
4. 병렬 연산에 적합하도록 클래스 구조 **모듈화 및 리팩토링**  
5. 멀티스레드 적용 후 기존 결과와의 **정합성 검증** 및 **성능 향상 정량 평가**
