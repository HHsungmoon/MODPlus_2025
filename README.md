# MODPlus_2025

## 🧪 프로젝트명

**MODplus의 비제한적 탐색 (Unrestrictive Search) 성능 향상을 위한 멀티스레드 구현**

---

## 📄 프로젝트 요약

소프트웨어 성능 최적화에 흥미가 있어 성능 개선 프로젝트를 졸업 프로젝트로 선정했다.
프로젝트의 목적은 기존의 코드를 멀티스레드로 구현하여 **단축된 시간내에 동일한 결과가**나오도록 만드는 것이다.
해당 과제의 목적이 관심 분야와 일치해 프로젝트를 진행하게 되었으며, 앞으로 수행 시간을 **획기적으로 개선하여 대용량 데이터 분석에 적합한 구조로 리팩토링**하는 것이 목표이다.

기존 대비 **30배 이상의 실행 속도 개선**을 달성한 **최적화된 병렬 프로그램**으로, **오류 및 버그 없이 안정적으로 구현**하고자 한다.

---

## 1. 프로젝트 배경 및 목표

### ■ 프로젝트의 필요성
- 현재까지도 **2015년도 코드**를 사용 중이며, 단일 스레드 기반이라 **실행 속도가 매우 느림**
- **분석 데이터 규모는 계속 증가**하고 있으나, 기존 구조로는 처리에 한계 존재
- 멀티스레드 리팩토링을 통해 **본래 기능과 탐색 정확도를 유지**하면서도 수행 시간을 **대폭 단축** 가능
- 연구실 내 다양한 과제에서의 **실제 활용도 향상**이 기대됨
- github : https://github.com/HanyangBISLab/MODplus
---

### ■ 프로젝트의 정의
본 프로젝트는 단백질의 **Post-Translational Modification (PTM)** 분석을 효과적으로 수행할 수 있는 도구인 **MODplus**의 성능을 개선하기 위한 최적화 프로젝트입니다. Java로 구현된 기존 단일 스레드 기반 코드를 **멀티스레딩** 구조로 재구현하여 성능을 최대한 향상시키는 방식으로 리팩토링을 진행합니다.

특히 **Unrestrictive Search** 방식에서의 성능 병목 현상을 해결하기 위해 알고리즘 최적화, 멀티 스레드 병렬화를 적용합니다.

---

### ■ 프로젝트 최종목표
- **Unrestrictive Search 수행 시간**을 획기적으로 개선
- **대용량 데이터 분석에 최적화된 구조**로 리팩토링
- **탐색 정확도 및 결과 정합성 유지**
- **CPU 개수에 비례한 성능 확장성 확보**

---

### ■ 세부목표
1. 기존 MODplus 코드 분석 및 멀티스레드 적용 가능한 부분 식별  
2. Java의 `Thread`, `ExecutorService`, `ForkJoinPool` 등을 활용한 병렬 처리 구조 설계  
3. 기존 클래스 및 함수 최적화, 적절한 자료구조 적용  
4. 병렬 연산에 적합하도록 클래스 구조 **모듈화 및 리팩토링**  
5. 멀티스레드 적용 후 기존 결과와의 **정합성 검증** 및 **성능 향상 정량 평가**
### ■ 프로젝트의 최종목표와 세부목표를 달성하기 위해 필요한 제반사항

#### ✅ 단계별 수행 계획

1단계: **기존 코드 분석 및 설계 리팩토링**  
- 전체 클래스 구조 및 함수 호출 흐름 분석  
- 탐색 알고리즘 및 PTM 관련 연산 최적화  
- 병렬화 구조 설계

2단계: **멀티스레드 구조 도입**  
- `ExecutorService` 또는 `ForkJoinPool` 기반 병렬 로직 구현  
- 계산 중심 로직의 병렬화를 통한 탐색 시간 단축

3단계: **알고리즘 및 자료구조 최적화**  
- 선형 탐색/정렬 → 해시 기반 탐색 또는 최신 자료구조로 개선  
- 중복 계산 제거 및 **캐싱 전략** 도입  
- 연산 효율성 극대화

4단계: **정합성 검증 및 성능 테스트**  
- 기존 버전과 동일한 결과 산출 여부 확인  
- CPU 수에 따른 처리 속도 향상 비율 측정 (예: 1, 8, 32, 36 Core)  
- **벤치마크 기록 수행**

5단계: **코드 정리 및 문서화**  
- Deprecated 코드 제거  
- 리팩토링 전후의 구조 차이 및 성능 비교 문서화  
- 구현 기능 문서 정리

### ■ 기대되는 프로젝트 최종 결과물

- 최종 산출물 **‘MODplus 2.0’**은 기존 대비 **30배 이상의 실행 속도 개선**을 달성한 **멀티스레드 기반 프로그램**입니다.
- 대규모 데이터를 빠르고 정확하게 분석할 수 있어 **생명공학 연구에 실질적 기여**가 예상됩니다.
- 분석 속도 개선을 통해 **연구시간 단축 및 효율적 실험 진행**이 가능해집니다.
- 다양한 학술 및 산업적 환경에서 적용 가능  
- 성능 최적화된 병렬 분석 도구로서의 활용도 증대
---

### ■ 프로젝트 수행을 통한 개인 역량 향상

- 약 6개월 간의 대규모 코드 분석, 병렬 프로그래밍 설계, CPU/메모리 연산 최적화 등을 통해 **실무 수준의 역량 강화**를 기대합니다.
- 특히 다음과 같은 능력의 향상을 목표로 합니다:
  - 멀티스레드 및 병렬 처리 구현 경험
  - 복잡한 코드 분석 및 구조 리팩토링 능력
  - 알고리즘 및 자료구조 최적화 설계 능력

> 이러한 역량은 향후 **대용량 데이터 처리 시스템**, **생물정보학 도구 개발**, **시스템 소프트웨어 분야** 진출 시 큰 경쟁력이 될 것입니다.
